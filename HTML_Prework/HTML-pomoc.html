Hiperłącza

Hiperłącza zewnętrzne - 'href': Przejdź do strony Google

Hiperłącza wewnętrzne "podstrona" - 'href': Przeniosę na stronę info.html

Hiperłącza wewnętrzne "kotwice" - 'href' + '#": Przejdź do spisu treści

przeniesie na stronę youtube.pl: .a href="http://youtube.pl". klik ./a.

przeniesie na podstronę kontakt.html: .a href="kontakt.html". klik ./a.

przeniesie do elementu o id równym "end": .a href="#end". klik ./a.

przeniesie na początek strony: .a href="#". klik ./a.

przeniesie do artykułu we wskazane miejsce: .a href="https://pl.wikipedia.org/wiki/HTML#Atrybuty". klik ./a.

atrybut 'rel':

  • wartość: 'dofollow' (podążaj) – domyślna .a href="http://www.coderslab.pl" rel="nofollow". nazwa_linka ./a.
  • wartość 'nofollow' (nie podążaj) .a href="http://www.coderslab.pl" rel="dofollow". nazwa_linka ./a.

atrybut 'target':

  • wartość: '_blank' – otwarcie strony w nowej karcie .a href="http://inna-strona.html" target="_blank". klik ./a.
  • wartość '_self' – otwarcie strony w tej samej karcie .a href="http://inna-strona.html" target="_self". klik ./a.
  • można dodać wartość 'noopener' .a href="http://inna-strona.html" target="_blank" rel="noopener". klik ./a.

pływająca ramka 'iframe': .iframe src="https://www.coderslab.pl" width="100%" height="400". ./iframe.

Identyfikatory i klasy

Atrybut 'id' czyli identyfikator: nadawany każdemu elementowi na stronie - pozwala odróżnić od siebie poszczególne elementy dokumentu HTML. Jest to unikalny identyfikator elementu, co oznacza, że dany id możemy przypisać tylko jednemu elementowi na stronie.
Przykłady:
< h3 id="smallHeader">Nagłówek z id < /h3>
< a href="http://google.com" id="link">
Link również może mieć swój identyfikator
< /a>
< p id="text">Paragraf z identyfikatorem< /p>

Klasy 'class': pozwalają odróżnić poszczególne elementy dokumentu HTML. Klasy idealnie przydają się do stylowania elementów.
Przykłady:
< h2 class="my-element"> < /h2>
< div class="my-element"> < /div>
< div class="my-element"> < /div>
< div class="my-element"> < /div>
< button class="btn btn-primary">Klik < /button>
< button class="btn btn-secondary btn-big">Klik < /button>

CSS kaskadowe arkusze stylów

Struktura CSS:
p { //selektor wskazujący na elementy na stronie
color: red; //właściwości:wartości właściwości
font-size: 20px;
border: 2px solid blue;
}
Najczęściej wykorzystywane selektory to znaczniki, id i klasy.

Po złapaniu elementu selektorem nadajemy mu wygląd właściwościami:
/* wszystkie p */
p {
margin-bottom: 20px;
}
/* element o unikalnym atrybucie id="element" */
#element {
color: red;
padding: 20px 0;
}
/* wszystkie elementy o klasie class="test" */
.test {
color: blue;
margin: 20px 0;
border: 2px solid #ddd;
}

Stylowanie po id jest możliwe i działa tak jak po innych selektorach, ale nie jest to najlepsza praktyka.
HTML:
< p id="testP">
lorem ipsum sit dolor
< /p>
< div id="element">
lorem ipsum sit dolor
< /div>
CSS:
#testP {
color: red;
}
#element {
padding: 10px 0;
color: blue;
}

W przeciwieństwie do id, jedną klasę może mieć wiele elementów. W praktyce CSS oznacza to, że dane stylowanie możemy dać wielu elementom jednocześnie.
HTML:
< h2 class="content">
lorem ipsum dolor sit
< /h2>
< p class="content">
lorem ipsum dolor sit
< /p>
< p class="content">
lorem ipsum dolor sit
< /p>
CSS:
.content {
font-size: 16px;
font-family: sans-serif;
color: blue;
font-weight: bold;
margin: 20px 0;
border: 1px solid #ddd;
padding: 20px;
}

    Stylowanie można dołączyć do naszej strony na trzy sposoby:
  • stylowanie inline w atrybucie style elementu (inline) - nie zalecane
    • < div style="color: red; font-size: 20px; font-weight: bold;">
    Przykładowy styl
    < /div>
    Takie stylowanie może być akceptowalne, gdy chcemy ukryć dany element:
    < div style="display: none;"> < /div>
  • wewnątrz znacznika < style> (internal)
    • Znacznik style możemy umieszczać w sekcji head lub body. Zaleca się umieszczać go w sekcji head, dzięki czemu style szybciej zostaną wczytane i zaaplikowane na stronie:
    < !doctype html>
    < html lang="pl-PL">
    < head>
    ...
    < style>
    body {
    margin: 20px;
    }
    .element {
    width: 100px;
    height: 100px;
    border: 1px solid red;
    margin-bottom: 20px;
    }
    < /style>
    < /head>
    < body>
    < div class="element">< /div>
    < div class="element">< /div>
    < div class="element">< /div>
    < /body>
    < /html>
  • dołączenie oddzielnego pliku ze stylami (external)
    • Plik ze stylami dołączamy do strony używając znacznika link. Powinien on nosić rozszerzenie .css. Element < link> importuje zewnętrzne pliki na stronę – w tym przypadku importuje arkusz CSS. Składa się z następujących atrybutów:
    - Atrybut rel (relation, związek) określa, jakiego typu jest importowany plik, dla CSS będzie miał wartość stylesheet ( arkusz stylów): rel="stylesheet".
    - Atrybut href (hypertext reference, odniesienie w hipertekście) określa położenie importowanego pliku: href="style.css".
    HTML:
    < !-- stronaX.html -->
    < !doctype html>
    < html lang="pl-PL">
    < head>
    < title>Podstrona X< /title>
    < link rel="stylesheet" href="style.css">
    < /head>
    < body>
    < div class="element">< /div>
    < /body>
    < /html>
    CSS:
    /* plik style.css */
    body {
    background: #eee;
    margin: 0;
    padding: 30px;
    }
    .element {
    border: 2px solid #aaa;
    margin-bottom: 20px;
    background: #eee;
    padding: 20px;
    }

Zagnieżdzanie i dziedziczenie - jeżeli znacznik jest zagnieżdżony w innym znaczniku, to przejmuje on niektóre jego style. W praktyce, z właściwości używanych na co dzień dziedziczone są tylko te odpowiedzialne za tekst.
CSS:
.parent-element {
/* związane z tekstem - dziedziczone */
color: red;
font-size: 20px;
text-transform: uppercase;
letter-spacing: 5px;
/* nie związane z tekstem - nie dziedziczone */
border: 1px solid red;
background: #ddd;
margin: 10px;
padding: 20px;
}

< div class="parent-element">
< div>
< div>

< /div>
< /div>
< /div>

/* plik style1.css */
p {
color: #222;
}
/* plik style2.css */
p {
padding: 20px;
background: red;
}
