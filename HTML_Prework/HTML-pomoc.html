  • wartość 'nofollow' (nie podążaj) .a href="http://www.coderslab.pl" rel="dofollow". nazwa_linka ./a.

atrybut 'target':

  • wartość: '_blank' – otwarcie strony w nowej karcie .a href="http://inna-strona.html" target="_blank". klik ./a.

CSS kaskadowe arkusze stylów->

Struktura CSS:
p { //selektor wskazujący na elementy na stronie
color: red; //właściwości:wartości właściwości
font-size: 20px;
border: 2px solid blue;
}
Najczęściej wykorzystywane selektory to znaczniki, id i klasy.

Po złapaniu elementu selektorem nadajemy mu wygląd właściwościami:
/* wszystkie p */
p {
margin-bottom: 20px;
}
/* element o unikalnym atrybucie id="element" */
#element {
color: red;
padding: 20px 0;
}
/* wszystkie elementy o klasie class="test" */
.test {
color: blue;
margin: 20px 0;
border: 2px solid #ddd;
}

Stylowanie po id jest możliwe i działa tak jak po innych selektorach, ale nie jest to najlepsza praktyka.
HTML:
< p id="testP">
lorem ipsum sit dolor
< /p>
< div id="element">
lorem ipsum sit dolor
< /div>
CSS:
#testP {
color: red;
}
#element {
padding: 10px 0;
color: blue;
}

W przeciwieństwie do id, jedną klasę może mieć wiele elementów. W praktyce CSS oznacza to, że dane stylowanie możemy dać wielu elementom jednocześnie.
HTML:

lorem ipsum dolor sit
< /h2>

lorem ipsum dolor sit

lorem ipsum dolor sit

Box model->

Każdy element na stronie można traktować jak swego rodzaju pudełko – stąd określenie box model. Są pudełka, które dostosowują się do swojej zawartości i układają się obok siebie (inline), są też pudełka, które domyślnie zajmują całą szerokość i układają się jeden pod drugim. Układanie się elementów określa się jako flow dokumentu.

    Właściwość display może przyjąć wiele wariantów:
  • block element jest jeden w danej linii, można mu nadawać wymiary i marginesy,
  • inline elementy wyświetlają się w linii, nie można nadać im wymiarów i pionowych marginesów,
  • inline-block coś pomiędzy inline a block, układa się obok siebie w linii, a jedncześnie można mu nadać wymiary i marginesy,
  • none wyłącza wyświetlanie elementu wraz z elementami w jego wnętrzu.
Poza trzema głównymi typami mamy też bardziej "specjalne": list-item, compact, run-in, table, table-caption, table-cell, table-row, table-row-group, table-column, table-column-group, inline-table, flex, inline-flex, grid, inline-grid.
Przykład użycia 'display':
p {
display: block; /* na całą szerokość, jeden w linii */
}
div {
display: inline; /* tylko tyle ile potrzebuje kontent, wiele w jednej linii */
}
h1 {
display: none; /* element nie będzie wyświetlany */
}
Każdy element na stronie można uznać za swoiste pudełko, któremu możesz nadać pewne właściwości. Kilka z nich określa jego rozmiary:
  • margin odległość od otaczających go elementów (dla inline działają tylko poziome)
  • border obramowanie danego elementu
  • width i height określają jego wymiary (pamiętaj: nie działają dla elementów typu inline)
  • padding czyli odstęp wewnętrzny od krawędzi danego elementu

Width i height - domyślnie każdy element na stronie ma szerokość i wysokość ustawioną na auto. Dla elementów blokowych oznacza to „całą możliwą szerokość” (nie jest to równoznaczne z width 100%).
CSS:
.parent {
border: 1px solid #ddd;
}
.child1 { /* nie ustawiamy width więc ma ona auto */
height: 20px;
background: red;
margin: 10px;
color: #fff;
font-family: sans-serif;
}
.child2 {
height: 20px;
background: blue;
width: 100%; /* ustawiam width na 100% */
margin: 10px;
color: #fff;
font-family: sans-serif;
}

Margin - odpowiada za odstęp danego elementu od innych, otaczających go elementów.
Przykłady deklaracji:
CSS:
/* 'użycie właściwości specyficznych' dla każdej strony elementu oddzielnie*/
.element1 {
margin-top: 10px;
margin-right: 20px;
margin-bottom: 30px;
margin-left: 40px;
}
/* 'shorthand' - z każdej strony inny odstęp - góra, prawo, dół, lewo - jak wskazówki zegara */
.element4 {
margin: 10px 20px 30px 40px;
padding: 10px 20px 30px 40px; /* góra 10px, prawo 20px, dół 30px, lewo 40px */
}
/* połączenie technik - element ma 10px z każdej strony, ale z dołu 50px */
.element {
margin: 10px;
margin-bottom: 50px;
}
/* centrowanie w poziomie 'margin: 0 auto' */
.element {
width: 500px;
margin: 0 auto;
}
Shorthand - poza skróceniem zapisu, również nadpisuje (a w zasadzie cofa do stanu domyślnego) wcześniej użyte specyficzne właściwości danego typu.
CSS:
.element {
margin-bottom: 20px;
margin-top: 20px;
margin: 10px; /* powyższe właściwości zostały nadpisane/cofnięte do wartości domyślnych */
background-color: red;
background-image: url(...);
background-size: cover;
background: blue; /* nadpisaliśmy wszystkie powyższe zerując je do stanu początkowego */
font-family: sans-serif;
font-size: 20px;
font-weight: bold;
line-height: 2em;
font: 30px Arial, sans-serif; /* shorthand nadpisał powyższe właściwości dotyczące fonta */
}
Shorthand nie działa w drugą stronę, jeżeli nawet nie znamy konkretnej składni zapisu, możemy użyć skróconego zapisu, a następnie dodać konkretne rzeczy, których nam brakuje:
CSS:
.element {
margin: 10px;
margin-bottom: 30px;
font: 20px Arial, sans-serif;
font-weight: bold;
}
Z marginesami wiążą się 2 dziwne zachowania.
Pierwsze z nich to to, że pionowe marginesy dwóch elementów nie sumują się, a brany jest pod uwagę ten większy (tylko marginesy pionowe).


Drugie tyczy się elementów zagnieżdżonych w innych elementach. Domyślnie, jeżeli dziecko elementu ma margines, nie odsuwa on tego dziecka od krawędzi rodzica, a odsuwa samego rodzica.

Są na to rozwiązania. Wystarczy rodzicowi dać właściwość overflow: hidden lub dziecku dać padding zamiast marginesu.

}

}

< /div>

Border - opisuje, jak ma się zachować obramowanie elementu.
Składa się on z 3 właściwości, które musimy ustawić dla elementu, aby nadać mu border:
  • border-width właściwość ta określa grubość bordera, można ją nadawać dla całego elementu za pomocą właściwości border-width lub dla każdego boku z osobna:
    CSS:
    .element1 {
    border-width: 10px;
    }
    /* ta sama zasada co przy marginesach - kierunek wskazówek zegara */
    .element2 {
    border-width: 10px 20px 30px 40px;
    }
    .element3 {
    border-top-width: 10px;
    border-right-width: 20px;
    border-bottom-width: 30px;
    border-left-width: 40px;
    }
  • border-style ustawia kolor bordera, tak jak przy szerokości, kolor można ustawić zbiorczo dla wszystkich boków albo dla każdego boku indywidualnie:
    CSS:
    div {
    border-width: 10px;
    border-style: solid;
    width: 100px;
    height: 100px;
    margin: 10px;
    }
    .element1 {
    border-color: red;
    }
    .element2 {
    border-color: red blue green yellow;
    }
    .element3 {
    border-top-color: red;
    border-right-color: blue;
    border-bottom-color: green;
    border-left-color: yellow;
    }
  • border-color ostatnia właściwość wymagana do działania bordera to border-style, który określa styl obramowania:
    CSS:
    div {
    border-width: 2px;
    border-color: red;
    width: 100px;
    height: 100px;
    margin: 10px;
    }
    .element-1 {
    border-style: solid;
    }
    .element-2 {
    border-style: solid dotted dashed double;
    }
    .element-3 {
    border-top-style: solid;
    border-right-style: dotted;
    border-bottom-style: dashed;
    border-left-style: double;
    }
W większości przypadków takie długie deklaracje jak powyżej nie są używane, ponieważ są niepraktyczne. Zamiast nich stosowany jest skrócony zapis:
CSS:
.element1 {
border: 2px solid red;
}
.element2 {
border: 2px dashed blue;
}
.element3 {
border: 2px dotted green;
}
Powyższe przykłady pokazują specyficzną cechę rysowania borderów. Gdy obramowanie otrzyma różne kolory, krawędzie stykają się pod kątem – jak ramy obrazu. Można to wykorzystać w wielu sytuacjach jak np. rysowanie kształtów czy robienie grotów strzałek:
CSS:
.arrow1 {
width: 10px;
height: 10px;
border-top: 5px solid red;
border-right: 5px solid red;
transform: rotate(-45deg);
}
.arrow2 {
width: 0;
height: 0;
border: 10px solid yellow; /*transparent*/
border-bottom-color: red;
}
.arrow3 {
width: 0;
height: 0;
border: 10px solid yellow; /*transparent*/
border-bottom-color: red;
border-right-color: red;
}
Końcową wielkość boksa można obliczyć według wzoru:
realWidth = width + padding-left + padding-right + border-left-width + border-right-width
realHeight = height + padding-top + padding-bottom + border-top-width + border-bottom-width
Do takich rozmiarów trzeba jeszcze dodać marginesy, które odsuwają dany element od otaczających go elementów.

'Box-sizing' pozwala przestawić wyliczanie wielkości elementów w dwa tryby:
  • box-sizing: border-box - border i padding liczone są do środka elementu, rozmiar elementu się nie zmienia
  • box-sizing: content-box - border i padding liczone są na zewnątrz elementu, rozmiar elementu sie zmienia
Przykład:
CSS:
div {
width: 200px;
height: 200px;
background: red;
color: #fff;
margin: 20px;
}
.elementb1 {
box-sizing: border-box;
}
.elementb2 {
padding: 20px;
box-sizing: border-box;
}
.elementb3 {
padding: 20px;
border: 20px solid black;
box-sizing: border-box;
}
Takie wyliczanie rozmiarów elementów na stronach jest tak naturalne, że większość autorów w dzisiejszych czasach rozpoczyna swoje stylowanie od linijek:
* {
box-sizing: border-box;
}
'Overflow' pozwala kontrolować zachowanie elementu wobec niemieszczącej się zawartości. Możliwe wartości to:
  • hidden – zawartość niewidoczna poza elementem
    Wartość hidden oznacza: "przytnij wszystko, co wychodzi poza element":
    CSS:
    .element {
    padding: 20px;
    background: yellow;
    color: red;
    margin: 20px;
    width: 200px;
    height: 200px;
    overflow: hidden;
    }
  • auto – jeżeli treść się nie mieści pojawią się paski przewijania
    Wartość auto oznacza: "jeżeli zawartość się nie mieści, pokaż paski przewijania. Jeżeli zawartość się mieści, nie rób nic":
    CSS:
    .element {
    padding: 20px;
    background: yellow;
    color: red;
    margin: 20px;
    width: 200px;
    height: 200px;
    overflow: auto;
    }
  • scroll – paski przewijania są widoczne zawsze
    Wartość scroll oznacza: „pokaż paski przewijania – bez względu na to, czy zawartość się mieści, czy nie”:
    CSS:
    .element {
    padding: 20px;
    background: yellow;
    color: red;
    margin: 20px;
    width: 200px;
    height: 200px;
    overflow: scroll;
    }
  • visible – domyślna – widoczna poza elementem
  • inherit – dziedziczy zachowanie od rodzica
Kontrolowanie 'overflow' dla danych osi, możemy skorzystać z dodatkowych właściwości:overflow-x (pozioma) i overflow-y (pionowa):
CSS:
.element {
padding: 20px;
background: yellow;
color: red;
margin: 20px;
width: 200px;
height: 200px;
overflow: hidden !important; /* wymuszamy schowanie przewijania */
overflow-y: auto !important; /* jeżeli zajdzie potrzeba pojawią się paski na osi Y */
}
Przydatność overflow:
  • jeśli rodzicowi nadasz overflow: hidden, jego dzieci nie odsuną go swoimi marginesami
  • jeśli rodzicowi nadasz overflow: hidden, możesz nim naprawić problem z floatami
  • overflow: hidden może się przydać do ukrywania tekstu na elementach – np. gdy dany element ma tło z tekstem, który nie musi być widoczny w całości

Projektowanie formularzy->

Wszystkie pola formularza powinny być objęte elementem 'form', który w HTML służy do definiowania formularza.
< form>...< /form>
Znacznik form może przyjąć kilka atrybutów. Wśród nich wyróżniamy:

  • action – wskazuje na adres, na który zostanie wysłany formularz
  • method – wskazuje metodę, jaką zostanie wysłany formularz (get, post)
Przykład:
< form action="http://nasza-domena.pl/skrypt-na-serwerze.php" method="get">...< /for>
Elementy formularza - każdy formularz składa się z licznych elementów (kontrolek). Pozwalają one na pobieranie różnych danych od użytkownika.
Oto najpopularniejsze typy elementów:
  • input – pole tekstowe do wpisywania treści (może występować w kilku rodzajach)
  • radio – pole do wyboru jednej z opcji
  • checkbox – pole do wyboru kilku opcji
  • textarea – pole do wpisywania dłuższej treści
  • select – pole do wyboru opcji
  • range – pole do wyboru wartości z zakresu
  • button – przyciski w formularzach
Przykład zastosowania pól:
< form>
< label for="text">Pole tekstowe< /label>
< input type="text" id="text" />
< label for="email">Pole tekstowe< /label>
< input type="email" id="email" />
< label for="number">Pole tekstowe< /label>
< input type="number" id="number" />
< label for="date">Pole tekstowe< /label>
< input type="date" id="date" />
< label for="url">Pole tekstowe< /label>
< input type="url" id="url" />
< label for="password">Pole tekstowe< /label>
< input type="password" id="password" />
< input type="range" id="range" />
< input type="radio" id="radio" />
< label for="range">Pole radio< /label>
< input type="checkbox" id="chk" />
< label for="chk">Pole checkbox< /label>
< label for="textarea">Pole textarea< /label>
< textarea id="textarea">< /textarea>
< label for="select">Pole select< /label>
< select id="select">
< option>Opcja 1< /option>
< option>Opcja 2< /option>
< option>Opcja 3< /option>
< /select>
< button type="submit">Wyślij< /button>

 Label - jest to element, w którym opisujemy, co użytkownik powinien wpisać w dane pole. Elementu tego możesz używać na 2 sposoby.
W pierwszym z nich używamy atrybutu for, gdzie podajemy ID pola, którego dany label się tyczy:
< form>
< label for="name">Wpisz swoje imię< /label>
< input type="text" id="name" />
< /form>
 Drugi sposób polega na umieszczeniu tekstu i pola wewnątrz labela:
< form>
< label>
Wpisz swoje imię
< input type="text" />
< /label>
< /form>
 Atrybut name - każdy element w formularzu (uniklany w jego zakresie) powinien mieć atrybut name, czyli nazwę pola. Wartość danego pola zostanie wysłana na serwer właśnie pod taką nazwą:
< form>
< label for="name">Wpisz swoje imię< /label>
< input type="text" id="name" name="name" />
< label for="surname">Wpisz swoje nazwisko< /label>
< input type="text" id="surname" name="surname" />
< /form>
 Atrybut placeholder - element dodatkowo wyjaśniający użytkownikowi co i jak ma wpisać w dane pole. Tekst ten zniknie, gdy użytkownik wpisze coś do pola:
< form>
< label for="user">Wpisz swoje imię< /label>
< input type="text" id="name" name="name" placeholder="Wpisz swoje imię" />
< label for="surname">Wpisz swoje nazwisko< /label>
< input type="text" id="surname" name="surname" placeholder="Wpisz swoje nazwisko" />
< /form>
 Atrybut value - oznacza wartość danego pola, czyli to, co wybrał lub co wpisał w dane pole użytkownik:
< form>
< label for="user">Wpisz swój login< /label>
< input type="text" id="user" name="user" placeholder="Wpisz swój login" value="Paweł" />
< label for="pass">Wpisz swoje hasło< /label>
< input type="password" id="pass" name="pass" placeholder="Wpisz swoje hasło" value="tajnehaslo" />
< /form>
 Pole input - oznacza pojedynczą kontrolkę, którą będzie wypełniał użytkownik. Rodzaj inputa określa jego atrybut type, jest wiele rodzajów inputów:
  • text – pole tekstowe,
  • email – pole tekstowe do wpisywania e-maili,
  • number – pole do wpisywania tylko liczb,
  • date – pole do wpisywania lub wyboru daty,
  • url – pole tekstowe do wpisywania adresu www,
  • password – pole tekstowe do wpisywania haseł (wpisywany w to pole tekst jest zamieniany na kropki),
  • range – pole do wyboru wartości z zakresu,
  • color – pole do wyboru koloru.
< label for="text">Pole tekstowe< /label>
< input type="text" id="text" />
< label for="email">Pole email< /label>
< input type="email" id="email" />
< label for="number">Pole number< /label>
< input type="number" id="number" />
< label for="date">Pole date< /label>
< input type="date" id="date" />
< label for="url">Pole url< /label>
< input type="url" id="url" />
< label for="password">Pole password< /label>
< input type="password" id="password" />
< label for="range">Pole range< /label>
< input type="range" id="range" />
 Textarea - element ten służy do wpisywania wieloliniowej treści:
< label for="message">Pole textarea< /label>
< textarea id="message">< /textarea>
 Pole textarea jest wyjątkowym polem – nie ma atrybutu value. Jego wartość powinna pojawić się między otwierającym i zamykającym znacznikiem value:
< label for="message">Pole textarea< /label>
< textarea id="message">Poprzednia wiadomość wpisana przez użytkownika< /textarea>
 Jeżeli chcemy dla tego pola użyć atrybutu placeholder, znacznik otwierający i zamykający muszą znaleźć się obok siebie. Pomiędzy znacznikami nie może znaleźć się nawet "spacja":

< textarea id="message" placeholder="Wpisz wiadomość">< /textarea>

< textarea id="message" placeholder="Wpisz wiadomość"> < /textarea>
 Pole checkbox - charakterystyczny input, który służy do potwierdzania danego wyboru:
< form>
< label> < input type="checkbox" name="agreement" value="1" /> Wyrażam zgodę < /label>< br />
< label> < input type="checkbox" name="orderProduct" value="1" checked /> Zamawiam produkt < /label>
< /form>
Każde pole powinno mieć swoją wartość value która, jeżeli pole jest zaznaczone, zostanie przesłana na serwer. W powyższym przykładzie drugie pole dostało atrybut checked-dane pole ma być domyślnie zaznaczone oraz atrybuty name-obie wartości trafią na serwer pod innymi nazwami.
Pola checkbox mogą też występować w grupach o tej samej nazwie. Dzięki temu użytkownik może wybrać kilka opcji. W przypadku kilku pól o tej samej nazwie warto je pogrupować za pomocą znacznika < fieldset>, któremu nadajesz tytuł dla całej grupy za pomocą znacznika < legend>:
< form>
< fieldset>
< legend>Wybierz zwierzę< /legend>
< label> < input type="checkbox" name="pet" value="dog" /> pies < /label>< br />
< label> < input type="checkbox" name="pet" value="cat" /> kot < /label>< br />
< label> < input type="checkbox" name="pet" value="pig" /> świnka < /label>< br />
< label> < input type="checkbox" name="pet" value="hamster" /> chomik < /label>< br />
< label> < input type="checkbox" name="pet" value="chicken" /> kura < /label>
< /fieldset>
< /form>
 Pole radio - podobne do pola checkbox, różnica jest taka, że w danej grupie pól można jednorazowo wybrać tylko jedną opcję:
< form>
< fieldset>
< legend>Wybierz płeć< /legend>
< label> < input type="radio" name="gender" value="m" /> mężczyzna < /label>< br />
< label> < input type="radio" name="gender" value="w" /> kobieta < /label>
< /fieldset>
< /form>
 Pole select - służy do wybierania opcji z rozwijanego komponentu. Kolejne opcje, z których można wybierać, umieszczamy w znacznikach < option>. Każdy option powinien mieć swój atrybut value oraz tekst danej opcji, który będzie wyświetlany po rozwinięciu selecta:
< label for="city">Wybierz miasto< /label>
< select name="city" id="city">
< option value="-1">Wybierz< /option>
< option value="gdansk">Gdańsk< /option>
< option value="warsaw">Warszawa< /option>
< option value="wroclaw">Wrocław< /option>
< option value="zakopane">Zakopane< /option>
< /select>
 Domyślnie wybrana jakaś opcja, powinna otrzymać dodatkowy atrybut selected:
< label for="city">Wybierz miasto< /label>
< select name="city" id="city">
< option value="-1">Wybierz< /option>
< option value="gdansk">Gdańsk< /option>
< option value="warsaw" selected>Warszawa< /option>
< option value="wroclaw">Wrocław< /option>
< option value="zakopane">Zakopane< /option>
< /select>
 Button - przyciski w formularzach można tworzyć na dwa sposoby.
Jeden to zastosowanie elementu input, który otrzyma atrybut type="button":
< input type="button" value="Kliknij" />
 Drugim, zalecanym, to użycie znacznika < button>:
< button type="button">Kliknij< /button>
 Zalecenie to wynika z prostego faktu. Między otwierający i zamykający znacznik buttona można wstawiać html – np. span z ikonką wczytywania czy ikonką dla danego przycisku. W przypadku inputa, którego treść podawana jest w atrybucie value – nie ma takiej możliwości:
< button type="button">< strong>Kliknij< /strong> ten przycisk< /button>
 Button submit - przycisk do wysyłania formularzy typu submit (może to być input lub button), po kliknięciu takiego przycisku formularz zostanie wysłany:
< form>
...
< button type="submit">Wyślij< /button>
< /form>

Relacje między elementami HTML->

Elementy HTML tworzą pewną hierarchiczną, drzewiastą strukturę przypomninającą drzewo genealogiczne.
Elementem nadrzędnym jest znacznik < html>.
Między poszczególnymi elementami zachodzą relacje rodzic – dziecko, np. element < html> jest rodzicem elementów < head> i < body>. Struktura ta jest częścią modelu DOM (Document Object Model):

< !doctype html>
< html lang="pl-PL">
< head>
< title>Prosta strona< /title>
< /head>
< body>
< p>Treść paragrafu< /p>
< div>
< strong>Ważny tekst< /strong>
< em>Inny ważny tekst< /em>
< p>Następny paragraf< /p>
< /div>
< /body>
< /html>
